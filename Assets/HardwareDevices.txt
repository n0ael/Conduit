# Conduit Hardware-CC-Datenbank (Block L2, Masterplan)
#
# Format: [Geraetename] eroeffnet einen Geraete-Block, darin je Zeile
# "CC-Nummer = Parametername". Zeilen mit # sind Kommentare, Leerzeilen
# werden ignoriert. Diese Datei ist die FAKTOR-Bestueckung (im Programm
# eingebaut); eigene/fehlende Geraete gehoeren in eine GLEICH strukturierte
# Datei "HardwareDevices.txt" im Conduit-Einstellungsordner (neben
# GridSession.xml) -- die App liest beide und ergaenzt/ueberschreibt die
# Faktor-Daten pro Geraet (gleicher Geraetename = dasselbe Geraet, gleiche
# CC-Nummer = ueberschreiben, neue Nummer = ergaenzen).
#
# Quelle der CC-Nummern: jeweiliges offizielles Handbuch / midi.guide.
# Elektron Analog Heat nutzt AUSSCHLIESSLICH NRPN (14-Bit MSB/LSB-Paare),
# keine einfachen 7-Bit-CCs -- daher hier stellvertretend der Digitakt
# (dieselbe Elektron-CC-Philosophie, aber mit echten Plain-CCs).
